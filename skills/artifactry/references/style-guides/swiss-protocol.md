# Swiss Protocol

Swiss Protocol is a strict, monochrome, technical style for artifacts that need discipline, credibility, and exact hierarchy. Use it for product specs, founder memos, technical strategy, architecture briefs, internal RFCs, and concise pitch decks. It inherits from high-precision developer tools, black-white consumer systems, minimal mobility/product sites, and structured technical documentation.

## Core Idea

Reduce until only the necessary remains. The style should feel engineered, not decorated. A slide is successful when the reader can understand the structure before reading every word.

## Visual Signature

- Black, white, and gray as the main system.
- One optional accent used only for active state, warning, or selected idea.
- Hard grid, exact alignment, visible typographic scale.
- Thin rules, square or small-radius cards, very low shadow use.
- Large confident headlines with short body copy.
- Tables and diagrams that feel like specifications.

## Palette

Use true black, white, neutral grays, and a single accent if needed. Accent may be blue, red-orange, or violet depending on the artifact, but must stay scarce. Do not use gradients, pastel panels, or multiple brand colors.

## Typography

Use a modern grotesk or neo-grotesk. Font weights should be controlled: regular for display, medium for labels, bold only for compact emphasis. Mono can carry IDs, dates, file names, dimensions, and code fragments. Letter spacing stays at zero except tiny uppercase metadata.

## Layout System

Use a 12-column grid with strict baselines. Prefer asymmetry that is clearly intentional: large title occupying 7 columns, evidence in 5 columns; or a full-width rule separating sections. Use top-left anchors, page numbers, and exact margins. Do not center content unless the slide is a cover or statement page.

## Slide Recipes

- Cover: title, subtitle, date, one thin rule.
- Thesis: one sentence occupying most of the page.
- Spec table: requirement, rationale, owner, status.
- Decision matrix: options in columns, criteria in rows.
- Architecture: boxes, arrows, labels, and sequence numbers.
- Before/after: two large monochrome panels.
- Principle list: numbered rows with one sentence each.
- Closer: "Decision", "Next", "Owner", "Date".

## DOCX Translation

DOCX should look like a crisp internal memo. Use strong heading hierarchy, narrow tables, strict bullets, and minimal color. Include page headers and a clean table of contents when appropriate.

## Agent Rules

- Rewrite long prose into exact statements.
- Remove filler adjectives before designing.
- Use rules and spacing as the main visual devices.
- If a slide needs more than one accent color, the structure is probably weak.
- Validate that every text block fits without shrinking below professional size.

## Avoid

Avoid rounded SaaS cuteness, illustration, photographic collage, warm paper texture, glowing gradients, large decorative icons, and dense dashboard clutter.
