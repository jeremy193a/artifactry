# Playful Systems

Playful Systems is a friendly modular style for SaaS onboarding, internal tools, team workflow explainers, lightweight presentations, and approachable product education. It inherits from collaborative tools, automation products, colorful productivity systems, and friendly developer apps.

## Core Idea

Make complexity feel manageable. The style should be cheerful, structured, and operational. It is playful because it reduces friction, not because it adds random decoration.

## Visual Signature

- Soft white or warm neutral canvas.
- Modular cards, rounded blocks, tabs, labels, and small diagrams.
- A controlled multi-color palette used for categories or workflow states.
- Friendly icons or simple glyphs when they clarify function.
- Clear onboarding rhythm: step, example, action, result.

## Palette

Use neutral base plus 4 to 6 category colors. Colors should have defined roles: task, data, agent, output, warning, success. Avoid rainbow gradients without meaning. Use pastel fills with dark text for readability.

## Typography

Use a friendly sans with strong labels and readable body. Headlines can be medium-weight and direct. Mono appears for short commands, file names, or automation tokens.

## Layout System

Use cards, lanes, steps, boards, and small diagrams. Make repeated modules feel tactile and clear. Large cards should not float randomly; they should belong to a workflow, Kanban lane, stack, or grid.

## Slide Recipes

- Cover: promise plus 3 colorful output chips.
- Onboarding: install, choose style, export, validate.
- Workflow board: lanes with cards and owners.
- Automation map: trigger, action, condition, output.
- Feature grid: 2x3 modules with icons and short copy.
- Before/after: messy process vs structured agent workflow.
- Checklist: friendly rows with completion marks.
- Closer: next action and reusable command.

## DOCX Translation

DOCX should become an approachable guide. Use colored callouts, step tables, checklists, and example blocks. Keep it professional enough for internal distribution.

## Agent Rules

- Use color to encode meaning.
- Keep labels short and verbs active.
- Add icons only when they support scanning.
- Keep every module aligned to a grid.
- If content is serious or financial, choose another style.

## Avoid

Avoid childish illustration, random emoji, excessive rounded cards, saturated rainbow pages, and low-contrast pastel text.
