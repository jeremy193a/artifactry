# Performance Machine

Performance Machine is a precision-and-velocity style for automotive, sports, hardware, industrial, performance product, and high-stakes technical demo artifacts. It inherits from performance automotive systems, premium hardware surfaces, engineering spec sheets, and cinematic launch pages.

## Core Idea

Make the artifact feel fast, engineered, and controlled. Performance is communicated through precision, contrast, specs, and motion cues, not through clutter.

## Visual Signature

- Dark or stark white canvas with sharp high-contrast type.
- Velocity bands, spec strips, technical labels, and calibrated accent lines.
- Large object/image zones when product imagery exists.
- Sparse color: red, blue, yellow, or metallic accent depending on mood.
- Spec-sheet drama: numbers, dimensions, power, timing, capability.

## Palette

Use black, white, graphite, silver, and one performance accent. Accent can be red for intensity, blue for engineering, yellow/gold for premium energy. Do not combine multiple racing palettes unless the user explicitly asks.

## Typography

Use a strong sans with technical confidence. Headlines may be uppercase in short bursts, but avoid long all-caps paragraphs. Mono is excellent for specs, times, model IDs, and measurements.

## Layout System

Use asymmetry and strong diagonals sparingly. Layouts should feel aerodynamic: large object, spec rail, caption, supporting panel. Dense technical slides should still have one hero value.

## Slide Recipes

- Cover: object/product, large name, one spec.
- Spec sheet: 4 to 6 values in a calibrated strip.
- Performance comparison: two machines/options with identical structure.
- System cutaway: component diagram, labels, flow arrows.
- Timeline: acceleration, release, milestone, or process sequence.
- Risk/control: performance constraint and mitigation.
- Proof: benchmark, test, or field result.
- Closer: final capability statement.

## DOCX Translation

DOCX should become a product brief or technical datasheet. Use spec tables, comparison charts, captioned images, and controlled dark section dividers.

## Agent Rules

- Ask for real specs or clearly mark placeholders.
- Use numbers as design anchors.
- Keep motion cues structural, not decorative.
- Avoid fake horsepower-style claims.
- Validate image crops and spec readability.

## Avoid

Avoid overdone racing stripes, too many metallic effects, loud gradients, fake 3D parts, and aggressive all-caps text everywhere.
