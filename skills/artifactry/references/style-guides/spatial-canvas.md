# Spatial Canvas

Spatial Canvas is a collaborative mapping style for workshops, brainstorms, product discovery, process maps, strategy rooms, and visual planning artifacts. It inherits from infinite-canvas collaboration tools, whiteboard systems, colorful structured-data surfaces, and workshop-friendly productivity products.

## Core Idea

Turn messy thinking into a navigable board. The reader should understand clusters, relationships, sequence, and priority at a glance.

## Visual Signature

- Open canvas with visible zones, lanes, groups, and connector lines.
- Sticky-note panels, cards, tags, arrows, and dot clusters.
- Bright accent colors with semantic meaning.
- Hand-off between rough thinking and polished structure.
- Spatial hierarchy: cluster, node, connection, note, action.

## Palette

Use light neutral canvas with a controlled sticky-note palette: yellow, blue, green, pink, lavender, orange. Use each color as a category, not decoration. Use dark ink for all final text.

## Typography

Use friendly sans typography. Keep note text large and short. Use mono only for IDs or version markers. Titles can be bold but should not overpower the map.

## Layout System

Design around zones. Use swimlanes, quadrants, clusters, radial maps, or journey maps. Connector lines must be meaningful and not tangled. Leave enough whitespace between groups so the board can be read at social size.

## Slide Recipes

- Cover board: title plus 3 clusters preview.
- Journey map: stages across columns, notes inside each stage.
- Priority matrix: impact vs effort with plotted cards.
- System map: actors, tools, data, outputs.
- Brainstorm: grouped ideas with category labels.
- Decision board: options, criteria, chosen path.
- Workshop exercise: blank fields and prompts.
- Closer: action board with owners.

## DOCX Translation

DOCX should use tables, checklists, and embedded board images. If the visual board is central, export PNG/PDF alongside DOCX.

## Agent Rules

- Choose a spatial model before designing.
- Keep note text short enough to read.
- Use color only for categories, priority, or status.
- Avoid making every item equal if the content has hierarchy.
- Validate that connector lines do not cross important text.

## Avoid

Avoid random sticky notes, chaotic arrows, decorative doodles, illegible small handwriting fonts, and color with no semantic role.
