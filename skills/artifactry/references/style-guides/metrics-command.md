# Metrics Command

Metrics Command is a dense operational style for KPI packs, revenue updates, product analytics, trading summaries, growth reports, support reviews, and business intelligence exports. It inherits from analytics databases, monitoring dashboards, trading products, product analytics systems, and data-heavy enterprise surfaces.

## Core Idea

Every page should answer: what changed, why it matters, and what action follows. The style is dense, but not chaotic. Density must come from structured evidence, not from stuffing more text.

## Visual Signature

- Dashboard-like surfaces with metric strips, tables, charts, and status rows.
- Dark or light operational canvas depending on audience.
- Mono numerals and compact labels.
- High signal accents for change, anomaly, target, and risk.
- Cards arranged as operational modules, not decorative tiles.

## Palette

Use dark charcoal, white, cool gray, and a focused accent system. Green means positive movement, red means negative or risk, amber means attention, blue/violet means selected or active. Avoid using semantic colors as broad decorative backgrounds.

## Typography

Use sans for labels and explanations, mono for numbers. Large values should dominate when they drive the story. Labels are compact, uppercase only if legible. Body copy is minimal and often attached to a metric.

## Layout System

Use repeatable dashboard grids: 4-up metric strip, 2-column analysis, full-width table, right rail for notes, bottom action bar. Keep alignment precise. Tables need row rhythm, zebra or hairline separation, and clear column roles.

## Slide Recipes

- Cover: reporting period, 3 headline metrics, one summary sentence.
- KPI strip: target, actual, delta, owner.
- Funnel: stages with conversion rates and dropoff callouts.
- Incident/alert: anomaly value, suspected cause, next action.
- Cohort table: rows by segment, columns by metric.
- Trend: one simple chart, one annotation, one decision.
- Operating review: wins, risks, asks, blockers.
- Closer: action register with owner and deadline.

## DOCX Translation

DOCX should become an operating report. Use tables, compact headings, executive summary boxes, and action registers. Charts can be inserted as images if generated deterministically.

## Agent Rules

- Never invent data. If numbers are missing, create placeholders or ask.
- Promote the most decision-relevant metric, not the largest number.
- Use semantic colors consistently.
- Keep chart count low; choose charts only when they clarify trend or comparison.
- Validate that table text remains readable in PNG export.

## Avoid

Avoid decorative dashboards, fake charts, too many gauges, one-slide spreadsheet dumps, meaningless icons, and color-coded values without legends.
